도로별 센서 데이터와 시민 민원, 그리고 도로의 등급 정보를 결합하여 데이터 기반의 보수 우선순위(Priority Rank)를 도출합니다. 단순히 발생 횟수를 세는 것을 넘어, **도로의 특성**과 **사고의 치명도**를 고려한 가중치 시스템을 적용합니다.

### 산출 공식

모든 데이터는 세그먼트(`s_id`) 단위로 집계되며, 다음과 같은 가중치 시스템을 통해 최종 점수가 산출됩니다.

Score = (Total_Impacts X 1.0) + (Complaint_Count X 50.0) + (Road_Grade X 10.0)

---

### 1. 물리적 위험도 (Total Impacts - 1.0)

자동차 센서에서 감지된 물리적 충격의 총 횟수입니다.

- **로직 의미:** 도로 파손의 직접적인 증거입니다.
- **설계 의도:** 충격 감지는 기계에 의한 자동 수집이므로 데이터 양이 많습니다. 따라서 개별 충격의 가중치는 1.0으로 설정하여 기초적인 위험도 지표로 활용합니다.
- **고려 사항:** 동일 지점에서 단기간 내 충격이 급증할 경우, 포트홀이 급격히 심화되고 있음을 의미합니다.

### 2. 사회적 시급도 (Complaint Count - 50.0)

지자체나 앱을 통해 접수된 시민들의 민원 건수입니다.

- **로직 의미:** 실제 시민이 체감하는 불편함과 위험도를 상징합니다.
- **설계 의도:** 민원 데이터는 센서 데이터보다 희소하지만 **확정적인 위험 신호**입니다. 한 명의 시민이 직접 신고했다는 것은 이미 사고가 발생했거나 발생할 뻔했음을 의미하므로, **센서 충격 1회 대비 50배의 가중치**를 부여하여 우선순위를 비약적으로 높입니다.
- **고려 사항:** 민원 건수가 1건이라도 발생하면 점수가 최소 50점이 추가되어, 단순 충격만 있는 구역보다 빠르게 보수 리스트 상단에 노출됩니다.

### 3. 도로 위험 등급 (Road Grade - 10.0)

해당 도로의 행정적/물리적 등급을 나타냅니다.

- **로직 의미:** 도로의 성격(통행량, 속도 등)에 따른 잠재적 사고 치명도입니다.
- **설계 의도:** 도로 등급을 가산점으로 적용하여 중요도가 높은 도로(상위 등급)의 우선순위를 보정합니다. 정보가 없는 경우 기본값(3)을 부여하여 데이터 누락으로 인한 우선순위 왜곡을 방지합니다.
- **고려 사항:** 향후 고속도로나 전용도로와 같이 속도가 높은 도로 등급에 더 높은 배점(예: 등급 1일 때 가산점 상향)을 적용하여 안전사고 예방 효과를 극대화할 수 있습니다.
